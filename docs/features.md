# 功能總覽

## 先看整體

LTDJMS 目前把 Discord guild 中最常見的四類流程放進同一個 bot：經濟互動、商品交易、派單協作，以及 AI 頻道治理。

## 依角色看目前能做什麼

| 角色 | 目前能力 |
| --- | --- |
| 一般成員 | 查餘額、玩骰子遊戲、逛商店、兌換商品、建立法幣訂單、在允許頻道提及 Bot |
| 管理員 | 設定貨幣、管理餘額 / 代幣 / 遊戲、管理商品與兌換碼、配置 AI 與派單設定 |
| 護航 / 售後人員 | 透過私訊與面板流程接單、推進完單、承接售後案件 |
| 維運 / 開發者 | 啟動 bot 與資料庫、追蹤 callback / 履約 / 事件管線、維護 schema 與設定 |

## 功能分組

### 1. Guild 經濟與遊戲代幣

- 入口：`/currency-config`、`/user-panel`、`/admin-panel`、`/dice-game-1`、`/dice-game-2`
- 成員可以查看 guild 貨幣餘額、遊戲代幣與交易 / 兌換歷史
- 管理員可以調整成員餘額、代幣與遊戲設定
- 骰子遊戲會消耗遊戲代幣並發放貨幣獎勵
- 相關變更會透過事件機制更新面板與快取

### 2. 商品、商店與兌換碼

- 入口：`/shop`、`/user-panel`、`/admin-panel`
- 管理員可以建立商品、配置獎勵、產生兌換碼、查詢兌換狀態
- 成員可以用 guild 貨幣購買商品，或在個人面板中輸入兌換碼
- 商品可發放貨幣、遊戲代幣，或觸發外部履約 webhook
- 獎勵發放失敗時，貨幣購買流程會執行自動退款

### 3. 綠界法幣付款與付款後處理

- 入口：`/shop` 的法幣商品購買流程、`EcpayCallbackHttpServer`
- 系統可向 ECPay 建立超商代碼訂單，並將訂單編號、付款資訊與付款期限透過 Discord 傳給買家
- 建單私訊會提醒買家需在期限內完成付款，否則訂單將被自動取消
- 付款回推到達後，系統會驗證 / 解密 payload，並用冪等流程更新訂單狀態
- 首次付款成功後，系統會再私訊買家付款成功通知，並附上訂單編號
- 付款完成後可觸發：
  - 外部履約 webhook
  - 護航商品的管理員通知
- 若 callback 重送，不應造成重複履約或重複通知

### 4. 護航派單與售後

- 入口：`/dispatch-panel` 與管理面板中的護航定價 / 售後設定
- 管理員可以建立護航派單、設定售後人員、調整護航定價
- 護航者會在私訊中確認接單與推進流程
- 客戶可確認完單，之後可進入售後申請 / 接案 / 結案流程
- 售後案件與付款後通知都使用 claim 機制避免多人重複處理

### 5. AI Chat、頻道限制與 AI Agent

- 入口：在允許頻道提及 Bot，以及 `/admin-panel` 中的 AI 設定
- AI Chat 只會在允許的頻道 / 類別運作
- AI Agent 頻道啟用是另一套設定，與一般 AI 白名單分開維護
- Agent 模式目前可執行的 Discord 工具包含：
  - 建立 / 列出 / 刪除頻道、類別、角色
  - 查詢與修改頻道、類別、角色權限
  - 移動頻道
  - 搜尋、發送、編輯、pin、刪除訊息
- AI 回應可經過 Markdown 驗證、切段與格式整理

## 重要限制與守門條件

- 多數互動只支援 guild，不支援在 DM 直接執行 slash command
- `/currency-config`、`/admin-panel`、`/dispatch-panel` 需要管理員權限
- AI 不在白名單頻道時會直接忽略 mention
- 僅當 `APP_PUBLIC_BASE_URL` 與 `ECPAY_RETURN_URL` 都未設定時，付款回推服務才不會啟動
- 付款、派單、售後這三類流程都依賴狀態轉移與冪等控制，不能只看表面 UI 流程
