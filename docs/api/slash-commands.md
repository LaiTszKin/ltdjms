# Slash Commands 參考文件

本文件列出 LTDJMS Discord Bot 目前註冊的所有 slash commands，包含用途、權限需求、參數與範例。

指令定義可在 `src/main/java/ltdjms/discord/currency/bot/SlashCommandListener.java` 中找到，本文件是該程式行為的文件化版本。

## Discord API 抽象層

從 v0.12.0 起，所有指令處理器使用統一的 Discord API 抽象層，而非直接依賴 JDA API。這提供了：

- **解耦**：業務邏輯與 Discord API 實作分離
- **可測試性**：使用 Mock 實作進行單元測試，無需依賴 JDA
- **擴展性**：未來可輕鬆替換 Discord API 實作（如 Discord4J）

抽象層包含：
- `DiscordInteraction`：統一的互動回應介面
- `DiscordContext`：事件上下文提取介面
- `DiscordEmbedBuilder`：視圖元件建構器
- `DiscordSessionManager`：Session 管理介面
- `DiscordButtonEvent`/`DiscordModalEvent`：特定事件類型抽象

## 指令總覽

| 指令 | 權限 | 模組 | 說明 |
|------|------|------|------|
| `/currency-config` | 管理員 | 貨幣系統 | 設定或查看伺服器貨幣名稱與圖示 |
| `/dice-game-1` | 所有成員 | 小遊戲 | 玩摘星手（原骰子遊戲 1），依骰子點數獲得貨幣獎勵 |
| `/dice-game-2` | 所有成員 | 小遊戲 | 玩神龍擺尾（原骰子遊戲 2），順子／三條計分可獲得高額貨幣獎勵 |
| `/shop` | 所有成員 | 商店 | 瀏覽伺服器可兌換的產品列表 |
| `/user-panel` | 所有成員 | 面板 | 顯示個人面板（貨幣餘額、遊戲代幣、流水紀錄） |
| `/admin-panel` | 管理員 | 面板 | 顯示管理面板，用按鈕與表單管理餘額、代幣與遊戲設定 |

> **注意**：原有的 `/balance`、`/adjust-balance`、`/game-token-adjust`、`/dice-game-1-config`、`/dice-game-2-config` 指令已整合至 `/user-panel` 與 `/admin-panel`，不再作為獨立指令提供；相關業務邏輯仍存在於服務層與歷史 handler 中，供面板與測試重用。

## AI Chat 訊息功能

AI Chat 功能透過**提及機器人**觸發，而非使用傳統的 slash commands。

### 使用方式

在任何 Discord 頻道中提及機器人即可觸發 AI 回應：

```text
@LTDJMSBot 你好
```

### 權限

- **所有成員**都可以使用 AI Chat 功能
- 機器人需要在該頻道有「傳送訊息」權限

### 行為說明

1. **提及檢測**：系統會檢測訊息是否包含機器人提及（`@BotName` 或 `<@BotId>`）
2. **訊息提取**：移除提及部分後，提取使用者的實際訊息
3. **預設問候**：如果訊息為空（僅提及機器人），會使用預設問候語「你好」
4. **AI 回應**：呼叫 AI 服務生成回應
5. **訊息發送**：將 AI 回應發送到同一頻道
6. **長訊息分割**：如果 AI 回應超過 Discord 2000 字元限制，會自動分割為多則訊息

### 錯誤處理

當 AI 服務發生錯誤時，會顯示友善的錯誤訊息：

| 錯誤類型 | 使用者看到的訊息 |
|---------|-----------------|
| 認證失敗 | `:x: AI 服務認證失敗，請聯絡管理員` |
| 速率限制 | `:timer: AI 服務暫時忙碌，請稍後再試` |
| 逾時 | `:hourglass: AI 服務連線逾時，請稍後再試` |
| 服務不可用 | `:warning: AI 服務暫時無法使用` |
| 空回應 | `:question: AI 沒有產生回應` |
| 格式錯誤 | `:warning: AI 回應格式錯誤` |

### 配置

AI Chat 功能由管理員透過 `.env` 檔案配置：

```bash
# AI 服務配置
AI_SERVICE_BASE_URL=https://api.openai.com/v1
AI_SERVICE_API_KEY=your_api_key_here
AI_SERVICE_MODEL=gpt-3.5-turbo
AI_SERVICE_TEMPERATURE=0.7
AI_SERVICE_MAX_TOKENS=500
AI_SERVICE_TIMEOUT_SECONDS=30
```

### 限制

- **無對話歷史**：系統不保存對話歷史，每次請求都是獨立的
- **訊息長度**：單則訊息限制 2000 字元（Discord 限制），超過會自動分割
- **連線逾時**：AI 服務連線逾時設定為 30 秒（可配置），不限制推理時間
- **僅限伺服器**：AI Chat 僅在伺服器頻道中有效，私人訊息不會觸發

## `/currency-config` – 設定／查看伺服器貨幣

- **權限**：需具備 Administrator 權限
- **用途**：設定或檢視伺服器使用的貨幣名稱與圖示。

### 參數

| 名稱 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `name` | string | 否 | 貨幣名稱，例如 `金幣`、`鑽石` |
| `icon` | string | 否 | 貨幣圖示，可為標準 emoji (`💰`) 或自訂表情（例如 `<:coin:1234567890>`） |

> 若兩個參數都省略，指令會顯示目前伺服器的貨幣設定。

### 使用範例

```text
/currency-config name:金幣 icon:💰
```

**典型回應**

```text
✅ Currency configuration updated!
Name: 金幣
Icon: 💰
```

## `/dice-game-1` – 摘星手（原骰子遊戲 1）

- **權限**：所有成員
- **用途**：消耗一定數量的遊戲代幣，擲 5 顆骰子，依點數總和換算伺服器貨幣獎勵。
- **遊戲規則概要**：
  - 每局擲 5 顆骰子（點數 1–6）。
  - 每顆骰子的獎勵為：`點數 × 250,000`，合計為本局總獎勵。
  - 獎勵會自動加入玩家在該伺服器的貨幣帳戶。
  - 每次遊戲消耗的代幣數量可透過 `/admin-panel` 中的「遊戲設定管理」調整。
- **本地化**：
  - 若使用者的 Discord 語言設定為 `zh-TW`，遊戲結果訊息會顯示為繁體中文。
  - 其他語言設定會顯示英文訊息。

### 參數

- 無（代幣消耗依伺服器設定）

### 使用範例

```text
/dice-game-1
```

**典型回應（zh-TW 語系）**

```text
**摘星手結果**
骰子結果：:two: :four: :six: :three: :five:

總獎勵：💰 5,000,000 金幣
新餘額：💰 15,000,000 金幣
```

**典型回應（英文語系）**

```text
**Dice Game Results**
Rolls: :two: :four: :six: :three: :five:

Total Reward: 💰 5,000,000 Gold
New Balance: 💰 15,000,000 Gold
```

> 若玩家遊戲代幣不足，系統會提示所需代幣與目前代幣餘額。

## `/dice-game-2` – 神龍擺尾（原骰子遊戲 2）

- **權限**：所有成員
- **用途**：消耗一定數量的遊戲代幣，擲 15 顆骰子，依「順子」與「三條」組合計分，可獲得高額貨幣獎勵。
- **本地化**：
  - 若使用者的 Discord 語言設定為 `zh-TW`，遊戲結果訊息會顯示為繁體中文。
  - 其他語言設定會顯示英文訊息。

> 詳細加權計分邏輯實作於 `DiceGame2Service` 中，這裡僅描述行為層面：玩家消耗代幣，依骰子結果獲得一筆貨幣獎勵，並記錄在貨幣帳戶中。

### 參數

- 無（代幣消耗依伺服器設定）

### 使用範例

```text
/dice-game-2
```

**典型回應（zh-TW 語系）**

```text
**神龍擺尾結果**
骰子結果：:one: :two: :three: :four: :five: :six: ...

順子：💰 2,100,000 金幣
三條：💰 3,000,000 金幣（2 組）
基礎：💰 200,000 金幣

**總獎勵：** 💰 5,300,000 金幣
**新餘額：** 💰 6,300,000 金幣
```

> 若玩家遊戲代幣不足，同樣會顯示所需代幣與目前代幣餘額。

## `/shop` – 商店

- **權限**：所有成員
- **用途**：瀏覽伺服器中可購買的產品列表，並使用貨幣直接購買商品。商店頁面會顯示產品名稱、描述、價格與獎勵資訊，並支援分頁瀏覽多個產品。

### 參數

- 無

### 使用範例

```text
/shop
```

**典型回應（含貨幣價格）**

- 若商店中有可購買商品：

```text
🏪 商店

────────────────────────────────────

**1. VIP 會員**
💰 價格：1,000 貨幣
商品描述：獲得額外福利
獎勵：1,000 貨幣

────────────────────────────────────

**2. 每日登入禮**
💰 價格：500 貨幣
商品描述：連續登入 7 天領取
獎勵：50 代幣

共 2 個商品
```

- 若商店中沒有產品：

```text
🏪 商店
目前沒有可購買的商品
```

### 分頁導航

當產品數量超過一頁（預設 5 個）時，頁面底部會顯示分頁按鈕：

- 「⬅️ 上一頁」：瀏覽前一頁（第一頁時禁用）
- 「下一頁 ➡️」：瀏覽後一頁（最後一頁時禁用）

### 購買流程

當商店中有可購買商品（已設定 `currency_price`）時，會顯示「💰 購買商品」按鈕：

1. 點擊「💰 購買商品」按鈕
2. 從選單中選擇要購買的商品
3. 系統顯示購買確認介面，包含：
   - 商品名稱與價格
   - 您的貨幣餘額
   - 購買後餘額
   - 若餘額不足會顯示警示
4. 點擊「確認購買」完成交易
5. 系統扣除貨幣並發放獎勵（若有）

**餘額不足範例：**

```text
💰 確認購買

**商品：** VIP 會員
**價格：** 1,000 貨幣
**您的餘額：** 500 貨幣

⚠️ **餘額不足！**

[確認購買] [取消]
```

### 兌換碼兌換

除了直接購買，商品仍可透過兌換碼兌換：

1. 管理員在 `/admin-panel` 中建立產品並生成兌換碼
2. 管理員將兌換碼發放給成員
3. 成員使用兌換碼進行兌換

## `/user-panel` – 個人面板

- **權限**：所有成員
- **用途**：以 Embed 形式顯示成員在該伺服器的：
  - 貨幣餘額與貨幣名稱／圖示
  - 遊戲代幣餘額
  - 查看貨幣流水的按鈕
  - 查看遊戲代幣流水的按鈕

### 參數

- 無

### 使用範例

```text
/user-panel
```

**典型回應**

- 一個僅自己可見（ephemeral）的 Embed，包含：
  - 「貨幣」欄位：顯示目前餘額與貨幣名稱／圖示
  - 「遊戲代幣」欄位：顯示目前代幣餘額
  - 「💰 查看貨幣流水」按鈕
  - 「📜 查看遊戲代幣流水」按鈕

### 流水紀錄分頁

點擊流水按鈕後，Bot 會顯示分頁式的交易紀錄，每一筆紀錄包含時間與說明。流水頁面會包含：

- 「🔙 返回主頁」按鈕：返回個人面板主頁
- 「⬅️ 上一頁」按鈕：若有前一頁
- 「下一頁 ➡️」按鈕：若有下一頁

範例流水紀錄：

```text
2025-12-01 12:34 +10 🎮 (摘星手獎勵)
2025-12-01 12:30 -3 🎮 (摘星手消耗)
...
```

## `/admin-panel` – 管理面板

- **權限**：需具備 Administrator 權限
- **用途**：提供圖形化管理介面，透過按鈕、下拉選單與 Modal 來：
  - 調整成員貨幣餘額（原 `/adjust-balance` 功能）
  - 調整成員遊戲代幣餘額（原 `/game-token-adjust` 功能）
  - 調整摘星手 / 神龍擺尾的代幣消耗設定（原 `/dice-game-1-config` 與 `/dice-game-2-config` 功能）
  - **AI 頻道設定**（V016 新增）：管理 AI 功能的允許頻道清單

### 參數

- 無

### 使用流程概要

1. 管理員輸入：

   ```text
   /admin-panel
   ```

2. Bot 會回覆一個僅管理員可見的管理面板 Embed，並附上四個主要按鈕：
   - 「💰 使用者餘額管理」
   - 「🎮 遊戲代幣管理」
   - 「🎲 遊戲設定管理」
   - 「🤖 AI 頻道設定」（V016 新增）

3. 點擊「使用者餘額管理」會彈出表單（Modal），可輸入：
   - 使用者 ID
   - 調整模式（增加／扣除／設定為指定值）
   - 金額

4. 點擊「遊戲代幣管理」會彈出表單，可輸入：
   - 使用者 ID
   - 調整金額（可為正負）

5. 點擊「遊戲設定管理」會先顯示選單讓你選擇遊戲種類（摘星手或神龍擺尾），再顯示表單輸入新的代幣消耗數量。

6. 點擊「🤖 AI 頻道設定」（V016 新增）會顯示 AI 頻道設定頁面，包含：
   - 目前允許的頻道清單（若有）
   - 「➕ 新增頻道」按鈕
   - 「➖ 移除頻道」按鈕
   - 「⬅️ 返回」按鈕

#### AI 頻道設定詳解（V016）

**行為模式**：
- **無限制模式**（預設）：未設定任何頻道時，AI 可在所有頻道使用
- **限制模式**：設定允許頻道清單後，AI 僅在清單中的頻道回應

**新增頻道流程**：
1. 點擊「➕ 新增頻道」
2. 從選單中選擇要新增的頻道
3. 系統會驗證機器人在該頻道是否有發言權限
4. 成功後顯示確認訊息

**移除頻道流程**：
1. 點擊「➖ 移除頻道」
2. 從選單中選擇要移除的頻道
3. 成功後顯示確認訊息
4. 若清空所有頻道，自動恢復無限制模式

**錯誤處理**：
- 該頻道已在清單中：顯示「該頻道已在允許清單中」
- 機器人無發言權限：顯示「機器人在該頻道沒有發言權限」
- 頻道不存在：顯示「該頻道不存在於允許清單中」

## 錯誤訊息與一般原則

在多數指令中，LTDJMS 會將可預期的失敗情況（例如輸入無效、餘額／代幣不足、伺服器尚未設定貨幣）包裝成 `DomainError`，再透過 `BotErrorHandler` 轉成使用者友善的錯誤訊息。

這些錯誤訊息通常具有以下特點：

- 使用自然語言簡要說明問題原因（避免出現技術細節或 stack trace）
- 盡可能包含指令名稱與關鍵參數（例如金額、目標成員）
- 在適當情況下提供下一步建議，例如：
  - 提示管理員先執行 `/currency-config` 完成貨幣設定
  - 提示成員先取得足夠遊戲代幣（透過管理員或遊戲獎勵）
- 對於個人敏感資訊或管理操作，多以 ephemeral 訊息回覆，避免在公共頻道洩漏資訊

若你在使用某個指令時反覆遇到錯誤訊息，建議：

1. 先確認輸入參數是否符合本文件描述的限制與型別。
2. 若是權限相關錯誤，請確認自己是否具備 Administrator 權限或該指令允許的角色。
3. 若你是開發者，可以對照：
   - `docs/architecture/overview.md` 的錯誤處理流程
   - 對應模組文件（例如 `docs/modules/currency-system.md`、`docs/modules/game-tokens-and-games.md`）
   找出是哪個業務規則導致錯誤。
